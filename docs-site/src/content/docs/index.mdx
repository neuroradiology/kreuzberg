## Why Xberg

<CardGrid>
	<Card title="One engine for everything" icon="random">
		Feed documents, URLs, code, images, and audio into one API. Get clean Markdown, entities, code
		structure, embeddings, and enrichment without stitching libraries together.
	</Card>
	<Card title="96+ document formats" icon="document">
		PDFs, Office documents, images, email, archives, and academic publications all standardize to
		clean Markdown plus five other output formats.
	</Card>
	<Card title="OCR and transcription" icon="magnifier">
		Tesseract, PaddleOCR, and VLM models for scans; Whisper for audio and video. Confidence scores,
		language detection, and automatic fallback chains.
	</Card>
	<Card title="Structured LLM extraction" icon="laptop">
		Extract entities, JSON schemas, and structured data using local LLM servers (Ollama, LM Studio,
		vLLM) or 143+ remote providers — no custom prompting required.
	</Card>
	<Card title="Code intelligence & embeddings" icon="setting">
		Extract functions, classes, imports, and symbols across 306 languages. Vectorize with local ONNX
		models or provider-hosted embeddings for RAG pipelines.
	</Card>
	<Card title="15-language native SDKs" icon="translate">
		Python, TypeScript, Rust, Go, Java, C#, Ruby, PHP, Elixir, Dart, Swift, Zig, and C FFI — plus a
		CLI, REST API, MCP server, Docker, and WASM.
	</Card>
</CardGrid>

:::tip[Choosing between TypeScript packages]
**`@xberg-io/xberg`** — Native NAPI-RS bindings. Use for Node.js servers and CLI tools. Full
feature set at native performance.

**`@xberg-io/xberg-wasm`** — Pure WebAssembly. Use for browsers, Cloudflare Workers, Deno, Bun, and
serverless environments (cross-platform, 60–80% of native speed).
:::

## Quick example

<Tabs syncKey="lang">
<TabItem label="Python">

```python title="main.py"
from xberg import ExtractInput, extract

output = await extract(ExtractInput(kind="uri", uri="document.pdf"))
print(output.results[0].content)
```

</TabItem>
<TabItem label="TypeScript">

```typescript title="index.ts"
import { ExtractInputKind, extract } from "@xberg-io/xberg";

const output = await extract({
  kind: ExtractInputKind.Uri,
  uri: "document.pdf",
});
console.log(output.results[0].content);
```

</TabItem>
<TabItem label="Rust">

```rust title="src/main.rs"
use xberg::{extract, ExtractInput, ExtractionConfig};

let config = ExtractionConfig::default();
let output = extract(ExtractInput::from_uri("document.pdf"), &config).await?;
println!("{}", output.results[0].content);
```

</TabItem>
</Tabs>
